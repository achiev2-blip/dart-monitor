# 구조 우선 워크플로우

코드를 수정하기 전에 반드시 아래 순서를 따른다. 코드부터 건드리지 않는다.

## 1. 구조 파악 (Read)
- 관련 파일의 코드를 **전부** 읽는다 (일부만 읽지 않는다)
- 데이터 흐름을 **시작(크롤링/입력)부터 끝(화면/저장)까지** 추적한다
- 중간에 거치는 함수, 엔진, 저장소를 모두 나열한다

## 2. 구조 이해 (Understand)
- 파악한 구조가 **유저의 목적에 맞는지** 판단한다
- 중복이 있는가? 누락이 있는가? 불필요한 것이 있는가?
- 구조 이해 결과를 유저에게 보고하고 **확인을 받는다**

## 3. 구조 정리 (Design)
- 목적에 맞게 구조를 먼저 정리한다 (추가/제거/통합)
- 구조 변경 계획을 유저에게 제시하고 **승인을 받는다**

## 4. 코드 수정 (Code)
- 정리된 구조에 맞춰 코드를 수정한다
- 구조 변경과 무관한 코드는 건드리지 않는다
