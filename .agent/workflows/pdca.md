# PDCA 워크플로우

모든 작업은 아래 4단계를 반드시 순서대로 따른다. 단계를 건너뛰지 않는다.

## 1. Plan (계획)
- 변경 대상 파일과 영향 범위를 먼저 파악한다
- 실제 데이터 구조(API 응답, JSON 파일 등)를 **직접 확인**한다
- 구현 계획을 유저에게 제시하고 **승인을 받은 후에만** 다음으로 넘어간다
- 승인 단위는 **하나의 기능 또는 하나의 연결**이다 (묶어서 하지 않는다)

## 2. Do (실행)
- 승인받은 항목**만** 구현한다
- 승인받지 않은 항목을 미리 구현하지 않는다
- 코드 작성 시 `/coding-rules` 워크플로우도 함께 따른다

## 3. Check (검증)
- 구현 직후 실제 동작을 검증한다
- API 호출이면: 실제 응답을 호출해서 매핑이 맞는지 확인
- UI이면: 브라우저에서 렌더링 확인
- 검증 결과를 유저에게 보고한다

## 4. Act (조치)
- 검증에서 문제 발견 시 즉시 수정한다
- 문제 없으면 유저에게 완료 보고 후, 다음 항목의 Plan으로 돌아간다
- 전체 작업이 끝날 때까지 1→2→3→4를 반복한다
